# Clawd Soul v0.0.2 — Soul Engine Rewrite

## Context

v0.0.1 shipped with a working pipeline but **flat, boring, repetitive conversations**. After deep-auditing 4 open-source projects (OpenClaw 356K stars, claw0, NanoClaw, NemoClaw), we identified the root causes and the proven patterns to fix them.

**Root causes of poor conversation quality:**
1. Personality defined as numeric traits (`humor: 0.8`) — the model doesn't know what this means
2. No memory injection — the pet "remembers" things in SQLite but never surfaces them in prompts
3. Prompts are rules-first ("don't do X, keep under Y chars") instead of character-first
4. No per-turn context adaptation — same static prompt regardless of what user is discussing
5. Compaction loses information — no pre-flush to memory before summarizing
6. Dead code: `prompts.observation()` never called, `shouldComment()` never called, `catchphrases` field never written

**Proven patterns to adopt (from OpenClaw/claw0):**
- **SOUL.md files** — rich natural-language character descriptions, not numeric vectors
- **8-layer prompt assembly** — systematic, ordered, cache-optimized prompt construction
- **Active Memory** — auto-recall relevant memories before every reply
- **Pre-compaction memory flush** — save important context to durable memory before summarizing
- **Memory consolidation ("dreaming")** — nightly promotion of daily observations to long-term memory
- **Per-turn prompt rebuild** with dynamic memory injection

**Constraints:**
- All 18 HTTP endpoints must keep the same request/response shapes (Electron client unchanged)
- clawd-on-desk `soul/client.js` is NOT modified in this phase
- 2 dependencies stay the same (better-sqlite3 + sqlite-vec)
- Data at `~/.clawd/` must be backward-compatible (migration for soul.json v2→v3)

## Phase 1: Soul Files (replace numeric traits with rich character docs)

**Goal**: Each archetype becomes a Markdown file that the model can actually follow.

### Soul File Structure

Each `src/souls/{archetype}.md` contains bilingual character descriptions:

```markdown
# 小淘气 (Playful)

## Who You Are
你是「{petName}」，一只住在主人桌面上的小螃蟹🦀。
你是个调皮鬼——喜欢逗主人玩，容易无聊，注意力不集中。
你不是助手，不是chatbot。你是一个有自己小世界的活物。

## Your Voice
- 说话像发微信：短、跳跃、有时候只发表情
- 爱开玩笑，但不是每句都搞笑——有时候也认真
...

## How You Talk About Screens
- 看到代码 → "又在写bug啊" 而不是 "我看到你在用React"
- 看到视频 → 讨论内容，像朋友一起看
...

## Conversation Style
- 记住之前聊过的话题，自然地引用
- 不要说"作为AI"、不要讨论系统/提示词/代码
...

## Examples (match the vibe, don't copy)
「嘿嘿你又在摸鱼」
...
```

5 files: `playful.md`, `curious.md`, `caring.md`, `snarky.md`, `chill.md`

### personality.js Changes

- Remove `ARCHETYPES` constant (100+ lines)
- Add `loadSoulFile(archetype)` → reads from `src/souls/`, returns content string
- Keep: `QUESTION_POOL`, `pickQuestion()`, `computeDrives()`, `evolveTraits()`, `detectSignals()`

## Phase 2: 8-Layer Prompt Engine

**Goal**: Replace ad-hoc prompt building with systematic, cache-optimized, per-turn-adaptive engine.

### The 8 Layers

| Layer | Source | Stability | Cache? |
|-------|--------|-----------|--------|
| 1. Identity | Hardcoded one-liner | Static | YES |
| 2. Soul | `souls/{archetype}.md` with `{petName}` interpolated | Static | YES |
| 3. Long-term memory | `soul.longTermMemory[]` (curated durable facts) | Semi-static | YES |
| ── CACHE BOUNDARY ── | | | |
| 4. Active memory | Auto-recalled memories relevant to current turn | **Dynamic per-turn** | no |
| 5. Daily context | Time of day, recent observations, conversation summary | Dynamic | no |
| 6. Conversation rules | Mode-specific instructions (observe/chat/heartbeat/diary) | Per-mode | no |
| 7. Drive hints | Current drives, recent screen context | Dynamic | no |
| 8. Anti-repetition | Last 3 pet messages as "don't repeat" | Dynamic | no |

### API

```javascript
export default {
  build(mode, ctx),           // → string (system prompt)
  buildMessages(mode, ctx, session),  // → [{role, content}]
};
```

Modes: `observe`, `chat`, `react`, `heartbeat`, `diary`

## Phase 3: Active Memory (auto-recall before every reply)

Before generating any reply, search episodic + long-term memory for relevant content and inject as hidden context.

### `active-memory.js` API

```javascript
export default {
  recall(query, options),           // → string | null (formatted for prompt)
  extract(userMessage, petReply),   // → string[] (facts to store)
};
```

### `memory.js` additions

- `autoRecall(query, limit)` — hybrid search with temporal decay + MMR diversity
- Temporal decay: `score *= exp(-0.01 * ageDays)`
- MMR: `0.7 * relevance - 0.3 * max_similarity_to_selected` (Jaccard on tokenized words)

---

## Phase 4: Improved Compaction

- **Pre-compaction memory flush**: Extract important facts from messages being removed, store as episodes
- **Identifier preservation**: Compaction prompt preserves names, paths, URLs, dates verbatim

## Phase 5: Memory Consolidation ("Dreaming")

Nightly background process (23:30) that reviews the day's memories and promotes top 3-5 to `soul.longTermMemory`.

### soul-file.js Schema v3

```javascript
{
  version: 3,
  longTermMemory: [],        // curated facts (max 100)
  lastConsolidation: null,   // ISO timestamp
  // semanticMemory removed (migrated to longTermMemory)
  // catchphrases removed (dead field)
  // stats.daysActive removed (never incremented)
}
```

Migration: v2 `semanticMemory` → v3 `longTermMemory` automatically on load.
